# Repository Structure

This document describes the organization and purpose of directories and files in this repository.

## Directory Structure

```
Imputation/
├── data/                    # Data files (input and output)
├── docs/                    # Documentation
├── notebooks/               # Jupyter notebooks
├── reports/                 # Generated reports and visualizations
├── scripts/                 # Utility scripts
└── tools/                   # Additional tools (optional)
```

## Directory Descriptions

### `data/`
Contains all data files used in the analysis.

- **`data/raw/`**: Original, unmodified source data files
  - `aavail_customer_activity.csv`: Raw customer activity data
  
- **`data/processed/`**: Cleaned and transformed data files
  - `aavail_data_imputed.csv`: Dataset after imputation (generated)

**Note**: Raw data should never be modified. All transformations create new files in `processed/`.

### `docs/`
Project documentation and guides.

- `DATA_DICTIONARY.md`: Data schema and variable descriptions
- `PROJECT_PLAN.md`: Project methodology and analytical approach
- `PRESENTATION_APPENDIX.md`: Technical details for presentation
- `PRESENTATION_README.md`: RISE presentation usage guide
- `PRESENTATION_CHECKLIST.md`: Pre-presentation verification checklist
- `REFERENCES.md`: Academic and technical references (APA format)
- `VIEW_PRESENTATION.md`: Instructions for viewing HTML presentation
- `REPOSITORY_STRUCTURE.md`: This file

### `notebooks/`
Jupyter notebooks containing the analysis.

- `aavail_imputation_workflow.ipynb`: Primary imputation workflow and analysis
- `aavail_eda_assignment.ipynb`: Exploratory data analysis and visualizations
- `aavail_presentation.ipynb`: RISE slideshow presentation for deliverables

**Best Practices**:
- Run cells in order from top to bottom
- Clear outputs before committing (optional, for cleaner diffs)
- Document code cells with markdown explanations

### `reports/`
Generated reports, presentations, and visualization assets.

- `aavail_market_insights.html`: Executive dashboard (HTML)
- `aavail_eda_assignment.html`: EDA report export (HTML)
- `aavail_presentation.html`: Browser-viewable presentation (HTML)
- `view_presentation.html`: Presentation viewer page
- `assets/`: Directory containing all visualization images (PNG files)

**Note**: HTML files and assets are generated by notebooks. They can be regenerated by running the respective notebooks.

### `scripts/`
Utility scripts for project tasks.

- `create_simple_html.py`: Converts notebook to standalone HTML presentation
- `export_to_html.py`: Alternative export using nbconvert
- `export_presentation.sh`: Bash script for presentation export
- `verify_setup.py`: Verifies environment and dependencies

**Usage**: Run scripts from the repository root directory.

### `tools/`
Optional directory for additional tools or temporary files.

Currently empty, reserved for future use.

## Root Files

### Documentation
- `README.md`: Main project documentation and getting started guide
- `SETUP.md`: Detailed setup instructions
- `CONTRIBUTING.md`: Contribution guidelines
- `LICENSE`: MIT License

### Configuration
- `.gitignore`: Git ignore rules

## File Naming Conventions

- **Notebooks**: `aavail_<purpose>.ipynb` (e.g., `aavail_presentation.ipynb`)
- **Data files**: `aavail_<type>_<description>.csv` (e.g., `aavail_data_imputed.csv`)
- **Reports**: `aavail_<type>.html` (e.g., `aavail_presentation.html`)
- **Visualizations**: `<description>.png` (e.g., `churn_analysis.png`)
- **Scripts**: `<purpose>.py` or `<purpose>.sh` (e.g., `create_simple_html.py`)

## Best Practices

1. **Data Files**: Never modify files in `data/raw/`. Always create new files in `data/processed/`.

2. **Notebooks**: 
   - Run cells in order
   - Document your analysis with markdown cells
   - Clear outputs before committing (optional)

3. **Reports**: 
   - Regenerate HTML files by running notebooks
   - Keep visualization assets organized in `reports/assets/`

4. **Scripts**: 
   - Make scripts executable: `chmod +x scripts/script.sh`
   - Document script purpose and usage
   - Test scripts before committing

5. **Documentation**: 
   - Keep documentation up to date
   - Use clear, descriptive file names
   - Follow markdown formatting standards

## Adding New Files

When adding new files:

1. **Data**: Place in appropriate `data/` subdirectory
2. **Notebooks**: Add to `notebooks/` directory
3. **Documentation**: Add to `docs/` directory
4. **Scripts**: Add to `scripts/` directory
5. **Reports**: Add to `reports/` directory

Update this document if adding new directory structures or significant organizational changes.
